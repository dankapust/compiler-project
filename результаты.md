## Sprint 2 — Парсер и AST

### Как это работает

1. **Исходный код** (`*.src`) подаётся на вход
2. **Препроцессор** удаляет комментарии и раскрывает макросы
3. **Лексер** (Scanner) разбивает текст на токены (Sprint 1)
4. **Парсер** (Parser) строит AST из потока токенов методом рекурсивного спуска
5. **Визуализация** — AST можно вывести в текстовом, DOT (Graphviz) или JSON формате

### Формальная грамматика (GRAM-1 — GRAM-6)

| Требование | Статус | Как реализовано |
| --- | --- | --- |
| **GRAM-1** | Выполнено | `src/parser/grammar.txt` — полная EBNF грамматика |
| **GRAM-2** | Выполнено | Start symbol `Program`, нетерминалы для всех конструкций, терминалы = TokenType из Sprint 1 |
| **GRAM-3** | Выполнено | 9 уровней приоритета: Primary → Unary → Multiplicative → Additive → Relational → Equality → LogicalAnd → LogicalOr → Assignment |
| **GRAM-4** | Выполнено | Block, VarDecl, Assignment, If/Else, While, For, Return, Call, EmptyStmt (`;`) |
| **GRAM-5** | Выполнено | FunctionDecl (`fn name(params) -> type { ... }`), VarDecl, StructDecl |
| **GRAM-6** | Выполнено | Left-assoc: `+ - * / % && \|\|`; Right-assoc: `= += -= *= /= %=`; Non-assoc: `== != < <= > >=` |

### Парсер (PAR-1 — PAR-5)

| Требование | Статус | Как реализовано |
| --- | --- | --- |
| **PAR-1** | Выполнено | `Parser(tokens)`, `parse() -> ProgramNode`, ошибки с line/column, интеграция с лексером |
| **PAR-2** | Выполнено | Рекурсивный спуск: `_declaration()`, `_statement()`, `_expression()`, `_assignment()` → `_primary()`. Хелперы: `_match()`, `_consume()`, `_peek()`, `_is_at_end()` |
| **PAR-3** | Выполнено | Ошибки с позицией и ожидаемым токеном. Обработка пропущенных `;`, несовпадающих скобок. Panic-mode recovery (`_synchronize()`) |
| **PAR-4** | Выполнено | 1-token lookahead. Disambiguation: `_is_type_start()` проверяет `type_keyword` или `IDENTIFIER IDENTIFIER`. Dangling else — `else` привязывается к ближайшему `if` |
| **PAR-5** | Выполнено | O(n) для LL(1) грамматики. Память пропорциональна размеру AST |

## Команды для проверки

```bash
# Лексер
python -m cli lex --input examples/hello.src --output tokens.txt

# Парсер (текст)
python -m cli parse --input examples/hello.src --ast-format text

# Парсер (DOT для визуализации)
python -m cli parse --input examples/hello.src --ast-format dot --output ast.dot

# Парсер (JSON)
python -m cli parse --input examples/hello.src --ast-format json

# Парсер с verbose (ошибки + метрики)
python -m cli parse --input examples/hello.src --ast-format text --verbose

# Все тесты
python -m tests.test_runner

# Только парсер
python -m tests.test_runner --only parser
```
